# 产品契约

## 目标

Web 管理后台让单用户通过浏览器完成配置编辑、静态校验、运行时预览、生成预览和配置下载，同时保持 YAML 真相源不变。

核心目标：

- 用表单和预览降低 YAML 手写错误。
- 保持配置书写顺序和面板顺序一致。
- 明确区分草稿配置、已保存配置和当前运行时配置。
- 通过结构化诊断让用户能直接定位并修复问题。

## 用户与场景

主要用户是自托管使用者和开发调试者。典型场景：

- 编辑订阅、Snell、VLESS 和自定义代理来源。
- 调整过滤正则、节点组、服务组和规则集顺序。
- 保存前运行 `Prepare` 静态校验。
- 通过预览接口检查真实节点、分组展开和目标格式生成结果。
- 下载 Clash Meta 或 Surge 配置，或复制订阅链接给客户端。

## 范围

v2.0 Web 后台包含：

- A1-A8 配置编辑与静态校验页面。
- B1-B3 运行时预览、草稿生成预览和下载页面。
- C 系统状态页面。
- token 输入、只读配置源展示、revision 冲突提示、dirty 状态提示。

## 非目标

v2.0 Web 后台不包含：

- 多用户、RBAC 或审计系统。
- 服务端节点延迟探测图表。
- 内建配置版本仓库或一键恢复旧配置。
- 手机优先布局。最低支持分辨率为 1280x800。
- 直接在线编辑原始 YAML 文本。正式写回路径是结构化 JSON 到 YAML。

## 设计原则

- YAML 是真相源，UI 是可视化外壳。
- 前端草稿不等于运行时配置。
- 静态校验不证明生成可用，生成可用性由预览和生成路径验证。
- 配置源只读时，编辑页可查看但不可保存。
