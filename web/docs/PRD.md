# subconverter-go Admin · 产品需求文档 (PRD)

> 版本 v1.0 · 2026-04 · 配套设计稿：`subconverter Admin.html`

---

## 1. 背景

subconverter-go是一个通过单个 YAML 文件管理 Clash / Surge 节点订阅、分组、路由和规则集的工具。配置文件由人手工编写，写错了一个正则、一个 `proxy-providers` 路径，整份配置就拒绝加载。

本管理后台的目标是：**在不偏离 YAML 真相源的前提下，把"编辑 + 校验 + 热重载"流程变成一个可视化、低风险、可回滚的过程**。

## 2. 用户与场景

### 2.1 主要用户

| 角色               | 描述                          | 关键诉求                                                                          |
| ------------------ | ----------------------------- | --------------------------------------------------------------------------------- |
| **重度自托管用户** | 自建 VPS、机场转换、家用网关  | 把 6+ 个订阅源、自定义节点和复杂路由策略统一管起来，不想每改一行就 `vim + reload` |
| **小团队管理员**   | 给 5–20 人共享一份配置        | 改完想立刻看是否会断网；要能回滚                                                  |
| **开发者**         | 写 / 调试 subconverter 上下游 | 需要看到中间产物（节点池、分组成员、最终 Clash/Surge 输出），而不是只看 YAML      |

### 2.2 核心场景

1. **新增一个机场订阅** → 粘 URL → 看到节点立即出现 → 拖进 🇯🇵 分组 → 保存。
2. **修改路由策略**，比如把 Netflix 从美国分组切到日本分组 → 改完一键校验 → 没错才允许保存。
3. **节点筛选**：用正则把订阅里的「剩余流量」「广告」节点扔掉，看着实时匹配数。
4. **回滚**：刚才热重载失败，从历史里点回上个版本。
5. **生成配置**：随时下载 Clash / Surge 文件，或拿 webhook URL 直接被客户端拉取。

## 3. 信息架构

后台采用**单层左侧导航 + 三段分组**结构，与 YAML 顶层 key 一一对应：

```
配置管理 (A)        运行时预览 (B)      系统 (C)
├ A1 订阅来源        ├ B1 节点预览        └ C  系统状态
├ A2 过滤器          ├ B2 分组预览
├ A3 节点分组        └ B3 生成与下载
├ A4 路由策略
├ A5 规则集
├ A6 内联规则
├ A7 其他配置
└ A8 配置校验
```

- **A 区 = 编辑**：每一项对应一个 YAML 段，所见即所写。
- **B 区 = 真相**：运行时展开后的样子，校验逻辑也基于这一层。
- **C 区 = 健康**：进程、热重载历史、探针。

## 4. 功能模块

### 4.1 A1 订阅来源

按"协议族"分四类卡片：SS 订阅、Snell 池（仅 Surge）、VLESS 池（仅 Clash）、自定义代理。每张卡片显示 URL、节点数、实时连通状态、刷新/编辑/删除按钮。自定义代理支持中转链。

**关键交互**：

- 添加 → 居中模态（粘 URL → 自动识别协议 → 显示预览节点数 → 保存）
- 删除 → 居中确认弹窗（"删除订阅？该操作不可撤销"）
- 顶栏全局动作：`校验` / `保存并热重载`

### 4.2 A2 过滤器

正则 + 实时匹配预览。左侧是规则编辑器，右侧实时显示当前节点池里"会被过滤掉"的节点列表。

### 4.3 A3 节点分组

按地区/属性聚合节点。每个分组卡片显示：名称 + 正则 + 命中节点数 + 策略（select / url-test / fallback）。300ms 防抖正则匹配，不合法正则会即时报错。

### 4.4 A4 路由策略

把分组、特殊关键字（DIRECT / REJECT / @all）、规则集编排成「服务组」。可拖拽调整成员顺序。

### 4.5 A5 规则集

挂载远端规则列表（GeoIP、广告、流媒体等）。每条显示 URL、行数、最后更新时间。

### 4.6 A6 内联规则

直接写在配置里的零散路由规则，表格 + 优先级。

### 4.7 A7 其他配置

`fallback / base_url / 模板` 等基础设置，键值表单。

### 4.8 A8 配置校验

保存前的全面校验页。错误（必修）、警告（建议）、提示（可选）三段集中列出。每条带 `跳转 →` 按钮直达问题字段。

**关键交互**：

- 跳转 → 在原页面右侧滑出 Drawer，直接编辑该字段，不丢失校验上下文。
- 校验中 → 顶栏「重新校验」按钮转 Spinner、禁用。

### 4.9 B1 节点预览

把所有来源（订阅 / Snell / VLESS / 自定义）拉取到的真实节点合成一张表。可按来源类型 Tab 过滤。每行显示：连通状态、名称、协议、服务器、端口、来源、Surge/Clash-only 标签。

### 4.10 B2 分组预览

把每个分组按当前正则展开，显示真实命中的节点列表，未命中的分组红色标注。下半部分展示「服务组 → 分组 → 节点」的展开链路。

### 4.11 B3 生成与下载

可下载 Clash Meta、Surge 配置文件，或复制 webhook URL（带 token）。展示文件大小、节点数、更新时间。

### 4.12 C 系统状态

四张状态卡（服务状态 / 版本 / 配置加载状态 / 上次热重载），下方两列：左侧热重载历史（含每次的耗时、变更摘要、success/failed 状态），右侧运行环境（PID、端口、内存）和探针（/healthz、/generate、/reload 的状态码）。

## 5. 关键交互模式（已锁定）

经过一轮 demo 评审，从 15 个候选模式里锁定了这 5 种 pattern，**全后台统一遵守**：

| #   | 模式                      | 用于                               | 设计要点                                                                   |
| --- | ------------------------- | ---------------------------------- | -------------------------------------------------------------------------- |
| 1b  | **居中 Modal**            | 添加 / 编辑订阅、复杂表单          | 焦点高度集中，避免列表抖动；最大宽度 520px；ESC 关闭                       |
| 2a  | **Toast Success（右下）** | 保存成功、刷新成功                 | 4s 自动消失，左 4px 绿色色条 + 圆形对勾 + 标题 + 副标题（耗时 / 影响范围） |
| 2b  | **Toast Error（右下）**   | 保存/校验失败                      | 不自动消失，左 4px 红色色条，必带「查看详情」跳转                          |
| 3a  | **顶栏按钮 Spinner**      | 校验中 / 保存中                    | 仅替换触发按钮，禁用并加 spinner；其它操作仍可继续                         |
| 4a  | **居中 Confirm**          | 不可撤销操作（删除订阅、清空规则） | 红色感叹号图标 + 删除内容明确点名 + 红色「确认删除」按钮                   |
| 5b  | **A8 跳转 → 右侧 Drawer** | 校验报错跳修                       | 不离开校验页；Drawer 里直接改字段，关闭后回到错误列表，自动重校            |

不在以上之列的交互模式（顶栏长条 banner、内联确认、需打字确认等）已显式排除，避免风格漂移。

## 6. 非功能需求

| 维度   | 要求                                                               |
| ------ | ------------------------------------------------------------------ |
| 真相源 | 始终是一份 `config.yaml`，UI 只是它的可视化外壳                    |
| 热重载 | 保存后 < 200ms 完成；失败不替换上一份配置                          |
| 历史   | 至少保留最近 20 次配置版本，可一键回滚                             |
| 校验   | 离线校验，不需要联网；错误必须阻塞保存，警告不阻塞                 |
| 主题   | 浅色为主，预留深色（Tweaks 可切换）                                |
| 强调色 | 默认 Indigo `#6366f1`，可换成 Sky / Emerald / Amber / Red / Purple |

## 7. 暂不在范围

- 多用户 / 权限系统
- 节点延迟实时探测图表
- 移动端适配（最低支持 1280×800）
- 在线编辑 YAML（必须经表单）

## 8. 验收

- 13 个核心页面（A1–A8、B1–B3、C）在 1280×820 设计画布全部可看；
- 5 种交互模式可通过 Tweaks 演示状态切换实测；
- 浅色 / 深色 / 6 种强调色组合下视觉无破绽。
