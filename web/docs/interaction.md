# 交互契约

## 全局原则

- 操作反馈必须可定位、可重试、可关闭。
- 保存、reload 和预览状态必须区分。
- 错误提示必须提供下一步动作。
- 长耗时预览需要明确文案，不只显示 spinner。

## Modal

用于新增或编辑复杂条目。

- 居中显示。
- 表单校验在提交前完成基础格式检查。
- 取消不修改草稿。
- 保存只更新草稿，不直接写回后端，除非页面明确是提交动作。

## Toast

用于操作结果反馈。

- 成功 Toast 可自动消失。
- 失败 Toast 不自动消失。
- 失败 Toast 必须包含“查看详情”或跳转入口。

## Confirm

用于删除条目、丢弃草稿、复制含 token 的订阅链接等高影响动作。

- 文案必须点名影响对象。
- 危险操作使用独立 danger 样式。
- 首次保存本地 YAML 时必须提示注释、引号和格式风格可能丢失。

## Spinner 与禁用

- 触发中的主按钮显示 spinner 并禁用。
- 同一 mutation 不允许重复提交。
- 页面其他只读导航可以继续使用。

## 校验 Drawer

A8 诊断跳转使用右侧 Drawer 或等价的字段聚焦机制。

- 通过 `locator.json_pointer` 定位字段。
- 修改后更新草稿。
- 用户可重新校验。
- 若定位失败，显示父级 section 并保留原错误文本。

## revision warning

编辑期间发现配置 revision 变化时，顶栏展示 warning。该 warning 不自动覆盖用户草稿。
