# v3.0 部署原则

> 状态：v3.0 目标原则。本文描述目标部署约束。

## 总原则

v3.0 继续采用单二进制、低依赖、可容器化部署。架构打磨发生在内部模块边界和产品语义上，不以部署复杂度换取抽象纯度。

## 部署模型

- Go 后端托管 Web UI、后台 API、产物下载和健康检查。
- 生产环境默认同源访问，避免默认启用 CORS。
- 配置源支持本地文件、远程 URL 和导入内容，具体读写能力由 ConfigStore 声明。
- Web UI 与后端同版本发布。
- 默认不要求数据库、外部队列或多进程服务。

## 部署关注点

- RuntimeSnapshot 的 revision、加载时间和诊断应在状态接口中可观测。
- Diagnostic、日志和错误响应不得泄漏未脱敏 URL token。
- 导入导出必须遵守配置源和模板目录的读写边界。
- Resource Adapter 负责远程资源读取、缓存和脱敏。
- ConfigStore 负责配置 revision、只读能力和原子写入。

## 非目标

- 不引入数据库作为必选组件。
- 不要求反向代理才能使用管理后台。
- 不拆分为多个容器服务。
- 不维护多版本后台 API。
