# v3.0 信息架构

> 状态：v3.0 目标契约。本文描述 Web 工作台的信息组织方向，不代表当前页面已经调整。

## 总体结构

v3.0 Web UI 应围绕“配置工作台”组织。用户任务是页面组织的第一视角。

```text
配置工作台
  ├─ 来源与节点
  ├─ 分组与链路
  ├─ 路由与规则
  ├─ 目标格式
  ├─ 诊断中心
  ├─ 生成与导出
  └─ 系统状态
```

## 页面职责

### 来源与节点

管理订阅、Snell、VLESS 和自定义代理。重点展示来源状态、节点数量、格式专属性、过滤结果和拉取诊断。

### 分组与链路

管理地区组、链式组和匹配规则。重点展示匹配结果、空组风险、链式上游约束和 `@all` 语义。

### 路由与规则

管理服务组、rulesets、rules 和 fallback。重点展示引用关系、`@auto` 展开、路由环路和 fallback 可用性。

### 目标格式

展示 Clash Meta 与 Surge 的目标投影视图。重点回答：这个配置对某个目标格式是否可生成，哪些节点会被过滤，过滤后是否产生级联风险。

### 诊断中心

集中展示静态校验、运行时预览、目标投影和渲染诊断。诊断可以跳转回具体配置字段或目标格式页面。

### 生成与导出

生成目标格式配置、复制订阅链接、导出生效配置包、导入配置包。重点区分草稿配置、工作配置和运行时快照。

### 系统状态

展示版本、配置源、权限能力、runtime revision、dirty 状态、最近 reload 和运行环境。

## 导航原则

- 页面名称应对应用户任务，而不只是内部模块名。
- 每个页面应清楚说明数据来自草稿配置、工作配置还是 RuntimeSnapshot。
- 任何会实际拉取远程来源的操作都应提前提示。
- 诊断跳转应以 locator 为准，不依赖展示文本反查。
- 保存操作提交完整 Config DTO，并通过 revision 防止陈旧页面覆盖新配置。
